# 🎰 开奖逻辑详解

## 开奖流程概览

```
每日 21:00 IST 自动开奖
         ↓
1. 锁定奖池 (无法继续参与)
         ↓
2. 收集所有有效号码
         ↓
3. 加权随机抽取中奖号码
         ↓
4. 分配奖项等级
         ↓
5. 通知中奖用户
```

---

## 详细步骤

### 1️⃣ 奖池锁定
- **时间**: 21:00 IST
- **操作**: 奖池标记为 `locked: true`
- **目的**: 防止开奖期间有新充值进入

### 2️⃣ 收集有效号码
- **条件**: 当日生成的号码，`status: 'VALID'`
- **最低要求**: 至少10个号码才能开奖
- **不足处理**: 如果少于10个，不开奖，顺延到次日

### 3️⃣ 确定中奖人数
```javascript
winnerCount = min(总号码数 × 5%, 20)
```
- 中奖率: 5%
- 上限: 最多20个中奖者
- 示例: 100个号码 → 5个中奖者

### 4️⃣ 加权随机抽取

#### 权重系统
| 号码等级 | 权重 | 获得方式 |
|---------|------|---------|
| 🎁 Free | 1x | 注册/签到 |
| ⭐ Silver | 2-3x | ₹100-300充值 |
| ⭐⭐ Gold | 4-6x | ₹500-1000充值 |
| ⭐⭐⭐ Diamond | 6-9x | ₹2000-3000充值 |
| 👑 Crown | 8-15x | ₹5000-10000充值 |
| 👑👑 VIP | 18x | ₹20000充值 |

#### VIP加成
- VIP用户所有号码权重 × 1.5
- 示例: Gold号码权重4x → VIP用户为6x

#### 随机算法
```
总权重 = 所有号码权重之和

随机数 = 0 ~ 总权重 之间的随机值

累积权重遍历:
  号码1: 权重2 → 累积2
  号码2: 权重4 → 累积6
  号码3: 权重2 → 累积8
  
如果 随机数 ≤ 累积值 → 该号码中奖
```

**特点**: 权重越高，被抽中概率越大

### 5️⃣ 奖项分配

#### 奖金分配比例
| 奖项 | 人数 | 奖金比例 | 计算方式 |
|-----|------|---------|---------|
| 🥇 一等奖 | 1人 | 40% | 总奖池 × 40% |
| 🥈 二等奖 | 2人 | 各20% | 总奖池 × 20% |
| 🥉 三等奖 | 剩余 | 平分20% | 总奖池 × 20% ÷ 人数 |

#### 示例计算
假设:
- 总奖池: ₹10,000
- 中奖人数: 5人

分配:
- 一等奖 (1人): ₹10,000 × 40% = ₹4,000
- 二等奖 (2人): ₹10,000 × 20% = ₹2,000/人
- 三等奖 (2人): ₹10,000 × 20% ÷ 2 = ₹1,000/人

### 6️⃣ 号码状态更新

```
中奖号码 → status: 'WON'
未中奖号码 → status: 'USED'
```

### 7️⃣ 通知用户
- 中奖用户收到私信通知
- 包含中奖号码、奖项等级、奖金金额
- 提示48小时内联系管理员领奖

## 中奖概率计算

### 单号码中奖概率
```
单号码概率 = 该号码权重 / 总权重

示例:
- 用户A: 1个Free号码 (权重1)
- 用户B: 1个Gold号码 (权重4)
- 总权重: 1 + 4 = 5

用户A中奖概率: 1/5 = 20%
用户B中奖概率: 4/5 = 80%
```

### 多号码优势
```
用户拥有多个号码 = 多次抽奖机会

示例:
- 用户A: 10个Free号码 (每个权重1)
- 用户B: 2个Gold号码 (每个权重4)
- 总权重: (10×1) + (2×4) = 18

用户A中奖概率: 10/18 = 55.6%
用户B中奖概率: 8/18 = 44.4%
```

## 公平性保证

### 1. 随机数种子
```javascript
seed = Date.now().toString()  // 当前时间戳
```
- 不可预测
- 每次开奖不同

### 2. SHA256哈希
```javascript
crypto.createHash('sha256').update(seed).digest('hex')
```
- 密码学安全
- 无法操控

### 3. 权重透明
- 所有用户知道各等级权重
- VIP加成规则公开

### 4. 记录可查
- 中奖记录保存到数据库
- 可随时审计

## 管理员干预方式

### 正常流程
```
用户充值 → 获得号码 → 参与开奖 → 随机中奖
```

### 管理员可干预点

#### 1. 手动发号 (`/givenumbers`)
```
给指定用户发送号码
影响: 增加该用户中奖概率
```

#### 2. 手动添加中奖 (`/addtowinners`)
```
直接指定某用户中奖
影响: 绕过随机抽取
```

#### 3. 移除号码 (`/removenumber`)
```
删除指定号码
影响: 降低该用户中奖概率
```

⚠️ **警告**: 干预操作会记录日志，滥用可能导致信任危机

## 常见问题

### Q1: 为什么充值越多中奖概率越高？
**答**: 
- 充值多 = 号码多 = 抽奖机会多
- 高等级号码权重更高
- 但仍有运气成分，非100%中奖

### Q2: VIP有什么优势？
**答**:
- 所有号码权重 × 1.5
- 相同数量号码，中奖概率高50%

### Q3: 能否保证中奖？
**答**:
- 不能。这是概率游戏
- 权重高 ≠ 必然中奖
- 权重低 ≠ 不可能中奖

### Q4: 开奖结果能操控吗？
**答**:
- 技术上管理员可以干预
- 但所有操作有日志记录
- 建议保持公平，维护用户信任

## 技术细节

### 核心算法代码
```javascript
// 加权随机选择
weightedRandomSelect(candidates, count, seed) {
    const result = [];
    const temp = [...candidates];
    let random = this.seededRandom(seed);

    for (let i = 0; i < count; i++) {
        // 计算总权重
        const totalWeight = temp.reduce((sum, item) => 
            sum + (item.weight || 1), 0
        );
        
        // 生成随机数
        let r = random() * totalWeight;
        
        // 按权重选择
        let cumulative = 0;
        for (let j = 0; j < temp.length; j++) {
            cumulative += temp[j].weight;
            if (r <= cumulative) {
                result.push(temp[j]);
                temp.splice(j, 1);  // 移除已选中的
                break;
            }
        }
    }
    return result;
}
```
