## 一、奖池系统概述

### 1.1 设计目标
- **可持续性**：平台盈利 + 用户激励平衡
- **吸引力**：奖池足够大，吸引用户参与
- **公平性**：按权重分配，多劳多得
- **可控性**：可调整参数，适应运营需要

### 1.2 核心公式
```
奖池 = 基础保底 + 充值抽成 + 动态加成 - 运营成本

实际发放 = 奖池 × 发放比例 (通常80%)
平台留存 = 奖池 × 留存比例 (通常20%)
```

## 二、奖池计算模型

### 2.1 基础参数配置

```javascript
// config.js - POOL 配置
POOL: {
    // 基础保底金额（无论有无充值都有的最低奖池）
    BASE_AMOUNT: 2000,
    
    // 充值抽成比例（进入奖池的比例）
    COMMISSION_RATE: {
        REGULAR: 0.15,    // 普通用户: 15%
        TIER: 0.10,       // 等级用户: 10%
        VIP: 0.08         // VIP用户: 8%
    },
    
    // 周末加成
    WEEKEND_BONUS: {
        ENABLED: true,
        RATE: 0.30        // 周末额外+30%
    },
    
    // 安全边界
    MIN_POOL: 2000,       // 最低奖池 ₹2,000
    MAX_POOL: 50000,      // 最高奖池 ₹50,000
    
    // 发放比例
    DISTRIBUTION: {
        FIRST: 0.40,      // 一等奖: 40%
        SECOND: 0.20,     // 二等奖(2人): 20% (每人10%)
        THIRD: 0.20,      // 三等奖: 20%
        PLATFORM: 0.20    // 平台留存: 20%
    }
}
```

### 2.2 奖池计算公式详解

#### 公式1：基础奖池计算
```
BasePool = BASE_AMOUNT
         + (RegularRecharge × 15%)
         + (TierRecharge × 10%)
         + (VIPRecharge × 8%)
```

#### 公式2：周末加成
```
WeekendBonus = BasePool × 30%  (仅周末)
```

#### 公式3：最终奖池
```
FinalPool = min(max(BasePool + WeekendBonus, MIN_POOL), MAX_POOL)
```

## 三、动态调整机制

### 3.1 根据用户量调整

| 日活用户数 | 建议BASE_AMOUNT | 建议MAX_POOL |
|-----------|----------------|-------------|
| <50 | ₹1,000 | ₹5,000 |
| 50-200 | ₹2,000 | ₹10,000 |
| 200-500 | ₹3,000 | ₹20,000 |
| 500-1000 | ₹5,000 | ₹50,000 |
| >1000 | ₹10,000 | ₹100,000+ |

### 3.2 根据充值量调整抽成

```
如果日均充值 < ₹50,000:
    COMMISSION_RATE = 15% (提高抽成，保证奖池)

如果日均充值 ₹50,000 - ₹200,000:
    COMMISSION_RATE = 12% (平衡模式)

如果日均充值 > ₹200,000:
    COMMISSION_RATE = 10% (降低抽成，让利用户)
```

### 3.3 特殊活动加成

```javascript
EVENT_BONUS: {
    NEW_YEAR: { rate: 0.50, days: ["01-01"] },      // 新年+50%
    DIWALI: { rate: 0.50, days: ["11-12", "11-13"] }, // 排灯节
    WEEKEND: { rate: 0.30, days: ["Sat", "Sun"] },   // 周末+30%
    BIG_GAME: { rate: 0.40, trigger: "pool > 20000" } // 大奖池+40%
}
```

## 六、风控与安全

### 6.1 奖池上限控制

**为什么需要MAX_POOL？**
- 防止奖池过大，平台无法兑付
- 控制运营风险
- 鼓励资金循环利用

**建议上限设置：**
```
MAX_POOL = 日充值峰值 × 1.5

示例:
- 历史最高日充值: ₹100,000
- 建议MAX_POOL: ₹150,000
```

### 6.2 最低奖池保障

**为什么需要MIN_POOL？**
- 保证即使无充值也有吸引力
- 维持平台活跃度
- 用户体验底线

**保底资金来源：**
- 平台预存资金
- 历史结余
- 运营预算

### 6.3 异常监控

| 异常类型 | 触发条件 | 处理措施 |
|---------|---------|---------|
| 奖池异常高 | >MAX_POOL×2 | 暂停新充值，人工核查 |
| 奖池异常低 | <MIN_POOL×0.5 | 注入保底资金 |
| 发放异常 | 中奖率<1% | 检查算法，通知用户 |
| 资金缺口 | 应发>实存 | 延期发放，补足资金 |

## 七、运营策略建议

### 7.1 新平台启动策略

**Phase 1: 启动期 (1-2周)**
```
- BASE_AMOUNT: ₹2,000 (平台垫付)
- 大力推广，吸引首批用户
- 保证100%中奖率(参与即中奖)
- 目标: 日活50人
```

**Phase 2: 成长期 (1-2月)**
```
- 逐步减少保底比例
- 引入真实抽成模式
- 目标: 日活200人，盈亏平衡
```

**Phase 3: 稳定期 (3月+)**
```
- 正常运营模式
- 动态调整参数
- 目标: 稳定盈利
```

### 7.2 提升奖池吸引力

| 策略 | 实施方式 | 预期效果 |
|------|---------|---------|
| 周末大抽奖 | 周末奖池×1.5 | 周末充值+30% |
| 节日特别版 | 节日奖池×2 | 节日活跃+50% |
| 累进奖池 | 无人中奖则累积 | 刺激参与 |
| 保底大奖 | 一等奖最低₹1,000 | 降低参与门槛 |

### 7.3 风险控制策略

```
1. 资金储备
   - 保持3个月运营资金储备
   - 奖池资金专户管理

2. 分散风险
   - 单日奖池上限控制
   - 单用户中奖上限(如 ₹10,000)

3. 透明公开
   - 实时显示奖池金额
   - 公布历史开奖记录
   - 公示资金流向
```

## 十、附录

### 10.1 奖池计算代码

```javascript
/**
 * 计算今日奖池
 */
async function calculatePool(date) {
    // 获取今日充值数据
    const recharges = await getTodayRecharges(date);
    
    // 分类统计
    let regularAmount = 0;
    let tierAmount = 0;
    let vipAmount = 0;
    
    for (const r of recharges) {
        if (r.userType === 'VIP') {
            vipAmount += r.amount;
        } else if (r.userTier) {
            tierAmount += r.amount;
        } else {
            regularAmount += r.amount;
        }
    }
    
    // 计算抽成
    const commission = 
        regularAmount * CONFIG.POOL.COMMISSION_RATE.REGULAR +
        tierAmount * CONFIG.POOL.COMMISSION_RATE.TIER +
        vipAmount * CONFIG.POOL.COMMISSION_RATE.VIP;
    
    // 基础奖池
    let pool = CONFIG.POOL.BASE_AMOUNT + commission;
    
    // 周末加成
    if (isWeekend(date)) {
        pool += pool * CONFIG.POOL.WEEKEND_BONUS.RATE;
    }
    
    // 应用边界
    pool = Math.max(pool, CONFIG.POOL.MIN_POOL);
    pool = Math.min(pool, CONFIG.POOL.MAX_POOL);
    
    return Math.floor(pool);
}

/**
 * 分配奖金
 */
function distributePrize(pool, winners) {
    return {
        first: Math.floor(pool * CONFIG.POOL.DISTRIBUTION.FIRST),
        second: [
            Math.floor(pool * CONFIG.POOL.DISTRIBUTION.SECOND / 2),
            Math.floor(pool * CONFIG.POOL.DISTRIBUTION.SECOND / 2)
        ],
        third: Math.floor(pool * CONFIG.POOL.DISTRIBUTION.THIRD),
        platform: Math.floor(pool * CONFIG.POOL.DISTRIBUTION.PLATFORM)
    };
}
```

### 10.2 常见问题

**Q: 奖池可以固定不变吗？**
```
可以，设置:
- BASE_AMOUNT: 固定金额
- COMMISSION_RATE: 0
这样就变成固定奖池模式
```

**Q: 为什么我的奖池总是不够发？**
```
可能原因:
1. BASE_AMOUNT设置太低
2. COMMISSION_RATE太高
3. 充值抽成设置不合理
4. 平台留存比例过高

解决方案:
降低COMMISSION_RATE，增加BASE_AMOUNT
```

**Q: 如何实现累进奖池（无人中奖则累积）？**
```
修改代码:
if (winners.length === 0) {
    pool += yesterdayPool * 0.5;  // 50%累积
}
```
