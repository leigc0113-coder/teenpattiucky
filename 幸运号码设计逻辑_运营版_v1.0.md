## 一、号码系统设计概述

### 1.1 设计目标
- **公平性**：根据投入决定号码质量，高投入高回报
- **稀缺性**：限量号码创造收藏价值
- **激励性**：多层级设计激励用户持续参与
- **可控性**：管理员可灵活调整号码分配

### 1.2 核心原则
```
投入金额 = 号码数量 × 质量等级 × VIP加成
```

### 3.2 途径二：每日签到（Daily Check-in）

#### 签到规则
| 签到天数 | 号码等级 | 号码数量 | 说明 |
|---------|---------|---------|------|
| 第1-6天 | Free | 1个/天 | 普通签到奖励 |
| 第7天 | Silver | 3个 | 连续7天额外奖励 |
| 第8天起 | Free | 1个/天 | 重新计算周期 |

#### 连续签到算法
```javascript
function getCheckinReward(consecutiveDays) {
    if (consecutiveDays === 7) {
        return { tier: 'SILVER', count: 3, bonus: true };
    }
    return { tier: 'FREE', count: 1, bonus: false };
}
```

**示例场景**：
- 用户A：连续签到3天 → 获得 3个 Free 号码
- 用户B：连续签到7天 → 获得 3个 Free + 3个 Silver = 6个号码
- 用户C：第8天签到 → 重新计算，获得 1个 Free 号码

## 四、号码权重与中奖概率

### 4.1 权重计算原理

```
个人中奖概率 = 个人总权重 / 全场总权重

示例:
- 用户A: 3个Gold号码 = 12x 权重
- 用户B: 5个Silver + 2个Gold = 26x 权重
- 全场总权重: 1000x

用户A中奖概率: 12/1000 = 1.2%
用户B中奖概率: 26/1000 = 2.6%
```

### 4.2 不同等级号码的中奖表现

#### 模拟数据（1000次开奖）

| 用户类型 | 号码组合 | 总权重 | 预期中奖次数 | 实际收益 |
|---------|---------|--------|------------|---------|
| 散户A | 1 Free | 1x | 1-2次 | ₹40-80 |
| 散户B | 2 Silver | 4x | 4-6次 | ₹160-240 |
| 中产C | 4 Gold | 16x | 16-20次 | ₹640-800 |
| 大户D | 8 Crown | 64x | 60-70次 | ₹2400-2800 |
| VIP大户E | 8 Crown(VIP) | 96x | 90-100次 | ₹3600-4000 |

**结论**：
- 权重越高，中奖频率越高
- VIP加成显著提升收益
- 大额投资获得稳定回报

### 6.2 号码生成核心代码

```javascript
/**
 * 生成用户号码
 * @param {string} userId - 用户ID
 * @param {number} amount - 充值金额
 * @param {string} source - 来源类型 (recharge/checkin/invite/admin)
 * @param {string} date - 日期 YYYY-MM-DD
 * @returns {Object} 号码生成结果
 */
async function generateNumbers(userId, amount, source, date) {
    // 获取档位配置
    const tierConfig = getTierByAmount(amount);
    
    // 检查VIP状态
    const user = await UserService.getUser(userId);
    const vipMultiplier = user.isVIP ? 1.5 : 1;
    
    // 获取当日序列号起始值
    const startSeq = await getDailySequence(tierConfig.tier, date);
    
    // 生成号码
    const numbers = [];
    for (let i = 0; i < tierConfig.count; i++) {
        const seqNum = String(startSeq + i).padStart(4, '0');
        const numberId = `${tierConfig.code}-${seqNum}`;
        
        numbers.push({
            id: numberId,
            userId: userId,
            tier: tierConfig.tier,
            tierCode: tierConfig.code,
            baseWeight: tierConfig.weight,
            finalWeight: tierConfig.weight * vipMultiplier,
            source: source,
            date: date,
            createdAt: new Date().toISOString()
        });
    }
    
    // 保存到数据库
    await Database.insert('numbers', numbers);
    
    return {
        numbers: numbers,
        count: numbers.length,
        tier: tierConfig,
        totalWeight: numbers.reduce((sum, n) => sum + n.finalWeight, 0),
        vipMultiplier: vipMultiplier
    };
}

/**
 * 根据金额获取档位
 */
function getTierByAmount(amount) {
    const tiers = Object.keys(CONFIG.RECHARGE_TIERS)
        .map(Number)
        .sort((a, b) => b - a);
    
    for (const threshold of tiers) {
        if (amount >= threshold) {
            return CONFIG.RECHARGE_TIERS[threshold];
        }
    }
    
    return CONFIG.RECHARGE_TIERS[0];
}
```

## 八、常见问题（运营版）

### Q1: 如何调整各档位号码数量？
**A**: 编辑 `config.js` 中的 `RECHARGE_TIERS`，修改 `count` 值后重启服务。

### Q2: VIP加成倍数可以修改吗？
**A**: 可以，修改 `VIP.BONUS.weightMultiplier` 值，建议保持在1.3-1.8之间。

### Q3: 如何查看今日号码发放情况？
**A**: 管理员发送 `/stats` 查看各等级号码发放数量。

### Q4: 用户反映号码不够，可以临时加吗？
**A**: 可以，使用 `/givenumbers` 或 `/givetier` 命令手动补发。

### Q5: 号码系统出现故障如何应急？
**A**: 
1. 暂停新用户参与
2. 使用 `/drawnow` 测试开奖功能
3. 检查数据库连接
4. 联系技术团队
