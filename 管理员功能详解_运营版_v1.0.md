## 二、核心功能模块

### 2.1 充值审核系统

#### 2.1.1 审核流程图
```
用户提交截图
    ↓
系统自动通知管理员
    ↓
管理员查看审核队列
    ↓
查看截图 + Game ID
    ↓
判断:
  ├─ ✅ 通过 → 选择金额档位 → 发放号码
  └─ ❌ 拒绝 → 填写原因 → 通知用户
```

#### 2.1.2 审核通知格式
```markdown
🔔 New Recharge Pending Review!

👤 User: 123456789 (@username)
🎮 Game ID: 7654321
🆔 Ref: rc_abc12345
📅 Time: 2026-03-16 14:30 IST

📸 Screenshot: [图片]

💰 Select Amount to Approve:
[₹100]   [₹300]   [₹500]
[₹1000]  [₹2000]  [₹5000]
[₹10000] [₹20000]
[❌ Reject]
```

#### 2.1.3 审核操作按钮

| 按钮 | 金额 | 发放号码 | 适用场景 |
|------|------|---------|---------|
| ₹100 | ₹100 | 2 Silver | 小额测试 |
| ₹300 | ₹300 | 3 Silver | 小额充值 |
| ₹500 | ₹500 | 4 Gold | 标准档位 |
| ₹1000 | ₹1,000 | 5 Gold | 中额充值 |
| ₹2000 | ₹2,000 | 6 Diamond | 大额充值 |
| ₹5000 | ₹5,000 | 8 Crown | 大户档位 |
| ₹10000 | ₹10,000 | 10 Crown | 超大额 |
| ₹20000 | ₹20,000 | 12 VIP | 顶级用户 |

#### 2.1.4 审核标准指南

**✅ 通过标准：**
- [x] 截图清晰可见
- [x] Game ID 与截图一致
- [x] 充值金额可辨识
- [x] 无重复注册嫌疑
- [x] 无违规作弊行为

**❌ 拒绝原因：**
| 原因代码 | 说明 | 处理方式 |
|---------|------|---------|
| `BLURRY` | 截图模糊 | 要求重新上传 |
| `INVALID_ID` | Game ID错误 | 核实后重新提交 |
| `DUPLICATE` | 重复注册 | 永久拒绝并记录 |
| `FRAUD` | 作弊嫌疑 | 封号处理 |
| `WRONG_GAME` | 非目标游戏 | 引导正确游戏 |

### 2.5 开奖管理系统

#### 2.5.1 开奖操作

| 命令 | 功能 | 使用场景 |
|------|------|---------|
| `/drawnow` | 立即开奖 | 测试/特殊情况 |
| `/lockpool` | 锁定奖池 | 开奖前5分钟 |
| `/showwinners` | 查看中奖者 | 开奖后确认 |

#### 2.5.2 开奖流程

```
20:55 系统准备
  ├─ 锁定奖池（禁止新号码）
  ├─ 计算总权重
  └─ 生成中奖号码

21:00 自动开奖
  ├─ 抽取一等奖（权重随机）
  ├─ 抽取二等奖（2人）
  ├─ 抽取三等奖（剩余）
  └─ 记录中奖结果

21:01 通知用户
  ├─ 中奖者收到私信
  ├─ 群发开奖公告
  └─ 更新统计数据

48h内 发放奖金
  ├─ 确认UPI信息
  ├─ 转账到用户UPI
  └─ 标记为已支付
```

#### 2.5.3 手动开奖（紧急情况）

```
场景：系统故障或特殊活动

操作步骤：
1. /lockpool - 锁定当前奖池
2. /drawnow - 手动触发开奖
3. /showwinners - 查看结果
4. 核对无误后通知用户
5. 记录到系统日志

⚠️ 注意：手动开奖会记录操作者ID
```

## 四、常见问题处理

### 4.1 审核相关

**Q: 截图不清晰怎么办？**
```
1. 点击拒绝，选择原因 BLURRY
2. 系统自动通知用户重新上传
3. 用户重新提交后再次审核
```

**Q: 怀疑用户作弊？**
```
1. 暂不处理，标记待查
2. 使用 /finduser 查看用户历史
3. 检查是否有多个账号关联
4. 确认后使用 /ban 封号
```

**Q: 审核队列积压？**
```
1. 增加审核人手
2. 优先处理大额充值
3. 启用快速审核模式（小额自动通过）
4. 通知用户预计等待时间
```

### 4.2 技术相关

**Q: 系统无响应？**
```
排查步骤：
1. 检查 Railway 服务状态
2. 查看日志是否有错误
3. 确认数据库连接正常
4. 必要时重启服务
```

**Q: 开奖失败？**
```
应急处理：
1. 使用 /lockpool 锁定奖池
2. 手动执行 /drawnow
3. 核对中奖结果
4. 如仍失败，联系技术团队
```

### 4.3 用户投诉

**Q: 用户说没收到号码？**
```
处理流程：
1. /listnumbers GameID - 查询是否生成
2. 如已生成，检查是否被删除
3. 如确实缺失，使用 /givetier 补发
4. 记录补发原因
```

**Q: 用户质疑开奖公平性？**
```
回应模板：
"我们的开奖算法基于权重随机，所有号码和权重都是公开透明的。
您可以在 My Account 查看自己的号码权重。
如有疑问，我们可以提供开奖日志供您核实。"
```

```javascript

    timestamp: "2026-03-16T14:30:22Z",

    details: {
        amount: 500,
        gameId: "7654321"
    },
```

| 操作 | Level 1 | Level 2 | Level 3 |
