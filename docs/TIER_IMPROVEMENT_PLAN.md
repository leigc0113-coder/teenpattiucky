# 等级身份系统改进方案 v1.0

## 当前问题记录

### 现有设计
- 10个全球唯一等级号码 (Bronze·01 ~ Star·01)
- 纯累计充值升级
- 冷静期7天后号码释放

### 问题
1. 升级门槛太高 (Lv.10需要 ₹300,000)
2. 对印度用户不友好 (价格敏感市场)
3. 单一维度 (只看充值)
4. 大R体验差 (必须一级一级爬)

---

## 改进方案

### 方案1: 降低升级门槛 (推荐)

```
等级      名称            当前门槛      建议门槛     降幅
─────────────────────────────────────────────────────
Lv.1     Bronze·01      ₹0           ₹0          -
Lv.2     Bronze·02      ₹5,000       ₹1,000      -80%
Lv.3     Silver·01      ₹15,000      ₹3,000      -80%
Lv.4     Silver·02      ₹30,000      ₹6,000      -80%
Lv.5     Gold·01        ₹50,000      ₹10,000     -80%
Lv.6     Gold·02        ₹80,000      ₹20,000     -75%
Lv.7     Platinum·01    ₹120,000     ₹35,000     -71%
Lv.8     Platinum·02    ₹170,000     ₹50,000     -71%
Lv.9     Diamond·01     ₹230,000     ₹75,000     -67%
Lv.10    Star·01        ₹300,000     ₹100,000    -67%
```

### 方案2: 增加多维度升级

**活跃维度：**
- 连续签到7天 = 抵扣 ₹500 升级门槛
- 连续签到30天 = 直接送1个等级提升

**社交维度：**
- 每邀请1人注册 = 抵扣 ₹100 升级门槛
- 每邀请1人充值 ≥₹500 = 抵扣 ₹500 升级门槛

**活动维度：**
- 周末充值 = 双倍计入升级进度
- 节日活动 = 指定等级限时降低门槛

### 方案3: 高等级号码拍卖机制

**拍卖规则：**
- Star·01 / Diamond·01 等高等级号码定期拍卖
- 起拍价 = 对应等级门槛的 50%
- 拍卖周期 = 每周一次
- 时长 = 24小时
- 价高者得，立即获得号码

### 方案4: 号码挑战机制 (增加日活)

**挑战规则：**
- 拥有者48小时未登录机器人 → 号码进入"可被挑战"状态
- 挑战费：₹100
- 挑战方式：双方各抽1个号码比大小
- 胜者获得该等级号码
- 败者原号码进入冷静期
